# Software Engineering

Before we dive into software engineering, we have to clarify one thing: **software engineering does not involve coding alone.** As a student, I had always prioritized coding over concepts, but the "engineering" aspect is where the true depth lies. The purpose of this repository is to help me master software engineering fundamentals while I continue to sharpen my practical coding skills.

Now that we have addressed the elephant in the room, let's define what software engineering is. First, we have to break this down into two parts: **Software** and **Engineering**.

### **Software**
Think of software as the programs or instructions that tell a computer what to do. We generally categorize software into two types:

1.  **System Software:** This is software that controls electronic device hardware. A good example is the firmware on a USB flash drive. It contains a set of instructions so that when it is plugged into a computing device, the computer understands: *"This is a USB drive intended for file transfers."* Without this system software, the hardware would be unresponsive.
2.  **Application Software:** These are programs designed to help users perform specific tasks. They run on top of system software (such as an Operating System). A common example is Microsoft Word.

> If you are interested in a deeper dive into software types, check out this [GeeksforGeeks guide](https://www.geeksforgeeks.org/computer-science-fundamentals/software-and-its-types/).

### **Engineering**
When an **engineer** is mentioned, the word that should come to mind is **principles**. Engineering involves the application of scientific, mathematical, and creative principles to solve complex problems.

---

### **What is Software Engineering?**
Software Engineering is the disciplined process of applying engineering principles to the **design, development, testing, and maintenance** of software.

### **Who is a Software Engineer?**
Software Engineers are the professionals responsible for designing software systems. They apply engineering rigor to ensure that the software they build is reliable, efficient, and maintainable.
